ArchivoQué hace
src/preprocess.pyTransforma el CSV crudo de la EDSA en el dataset limpio con las 9 variables. Mismo pipeline del notebook.
src/train.pyEntrena los 5 modelos y registra métricas, parámetros y artefactos en MLflow automáticamente.
src/evaluate.pyCompara el nuevo modelo con el que está en producción y lo promueve si cumple los umbrales.
api/main.pyAPI REST con FastAPI. Endpoints para predicción individual, lote e información del modelo.
monitoring/monitor.pyDetecta drift en la distribución de IMC y clases. Genera reporte JSON con alertas.
run_pipeline.pyOrquesta todo: preprocess → train → evaluate → update baseline. Un solo comando para el ciclo anual.

🚀 Primer Uso (con EDSA 2023)

📋 Antes de empezar

Coloca el archivo datasetBoliviaSalud.csv (EDSA 2023) dentro de la carpeta data/ y renómbralo edsa_2023.csv.

Opción A — Pipeline completo (recomendado)

bash # Un solo comando para todo el proceso python run_pipeline.py --input data/edsa_2023.csv --year 2023 --model RandomForest # Resultado esperado: # ✓ Dataset procesado: 19,739 registros # ✓ 5 modelos entrenados y registrados en MLflow # ✓ RandomForest promovido a Production # ✓ Baseline de monitoreo actualizado

Opción B — Paso a paso

bash # Paso 1: Preprocesar python src/preprocess.py --input data/edsa_2023.csv --output data/processed/edsa_2023_procesado.csv # Paso 2: Entrenar los 5 modelos python src/train.py --data data/processed/edsa_2023_procesado.csv --year 2023 # Paso 3: Evaluar y promover el mejor modelo python src/evaluate.py --model RandomForest --year 2023

Ver los experimentos en MLflow UI

bash mlflow ui --backend-store-uri sqlite:///mlflow.db # Abrir en el navegador: http://localhost:5000 # Aquí verás todos los modelos, métricas y versiones

Iniciar la API

bash cd api && uvicorn main:app --reload --port 8000 # API disponible en: http://localhost:8000 # Documentación automática: http://localhost:8000/docs

📅 Ciclo Anual (nueva encuesta EDSA)

Cada año que el INE publica una nueva encuesta EDSA, ejecutas un solo comando:

bash # 1. Coloca el nuevo CSV en data/ # cp ~/Descargas/edsa_2024.csv data/edsa_2024.csv # 2. Ejecutar pipeline completo python run_pipeline.py --input data/edsa_2024.csv --year 2024 --model RandomForest # 3. Si el modelo fue promovido, recargar la API (sin reiniciarla) curl -X POST http://localhost:8000/modelo/recargar
⚠ Si el nuevo modelo NO mejora

El script lo indica y el modelo anterior se mantiene en producción automáticamente. No tienes que hacer nada más. Puedes forzar la promoción con --force si tienes razones justificadas.

python import mlflow client = mlflow.tracking.MlflowClient("sqlite:///mlflow.db") client.transition_model_version_stage( name="RiesgoMetabolico_RandomForest", version="1", # versión a restaurar stage="Production" )

📊 Monitoreo

bash # Ejecutar monitoreo sobre el dataset actual python monitoring/monitor.py --data data/processed/edsa_2023_procesado.csv # Salida esperada (sin alertas): # ✓ Drift IMC: OK # ✓ Drift Clases: OK # Actualizar baseline cuando entrenas con nueva encuesta python monitoring/monitor.py --data data/processed/edsa_2024_procesado.csv --update-baseline

El monitoreo detecta dos tipos de drift:

AlertaCondiciónQué hacer
Drift IMCMedia de IMC cambia más del 10% del baselineRevisar si hay cambio poblacional. Considerar reentrenar.
Drift ClasesProporción de Clase 2 cambia más del 15%Igual. Puede indicar cambio en hábitos alimentarios.
